# treecode v0.1 — Validated Design Spec

> Derived from `docs/PRD-treecode.md` (v0.1). This spec locks in the open questions and
> the v0.1 scope approved during brainstorming (2026-07-09). Where this spec and the PRD
> disagree, this spec wins for v0.1.

## Decisions locked in this spec

| Open question (PRD §14) | Decision |
|---|---|
| Drift state model | **Stateless.** Drift is recomputed from the real tree on every run. No lock file. Renames are inferred heuristically by content similarity. |
| Dependency-parsing aggressiveness | **Minimal.** Python via `ast`, JS/TS via import-statement regex. Everything else: empty `depends_on`. Parse failures never abort a run. |
| `InstructionsLoaded` logging | **Off by default**, opt-in via config. |
| Default generated language | **`en`**, overridable via `generated_language`. |
| v0.1 scope | **M1–M4 + minimal packaging.** Deferred to v0.2: `module-cartographer` subagent (M5), `--with-rules` (part of M6). Packaging (plugin.json, marketplace.json, README, LICENSE) is *in* scope because nothing is installable without it. `--generic` is in scope (it is a trivial config bypass). |

## Goal

A Claude Code plugin that generates and maintains a **distributed CLAUDE.md tree**:
a lean root map (< 80 lines) plus lazy, per-module nested CLAUDE.md files (30–60 lines),
written idempotently inside marker blocks, with drift detection and line-cap enforcement.

## Non-goals (v0.1)

- No duplication of auto memory (volatile learnings stay in `MEMORY.md`).
- No prose-quality linting (that is `claude-md-management` / `claude-md-improver`'s job).
- No auto-commit, ever. Generate → report → the user commits.
- No `.claude/rules/` generation (deferred with `--with-rules`).
- No parallel subagent exploration (deferred with `module-cartographer`).

## Invariants (from PRD §3 — non-negotiable)

1. Real nested CLAUDE.md files in subdirectories (lazy, loaded on access).
2. **Never** wire the tree with `@path` imports from the root (imports load at launch).
3. Idempotency via marker blocks; text outside markers is human-owned and never touched.
4. Root = map, not listing; root file stays under 80 lines.
5. Nested = 30–60 descriptive lines; no file listings.
6. Respect `.gitignore` / `.claudeignore`; never write into ignored/generated dirs.
7. Root survives compaction; nested files do not — never assume they are in context.

## Architecture

```
treecode/
├── .claude-plugin/
│   ├── plugin.json              # plugin manifest
│   └── marketplace.json         # catalog (marketplace "iu3qez-tools", source "./")
├── commands/
│   ├── map-tree.md              # /treecode:map-tree  (build/sync)
│   └── map-drift.md             # /treecode:map-drift (check-only, CI-friendly)
├── skills/
│   └── tree-mapper/
│       └── SKILL.md             # orchestrating workflow (lazy, on-demand)
├── hooks/
│   ├── hooks.json               # PostToolUse cap-guard + InstructionsLoaded log
│   └── cap-guard.sh             # thin wrapper invoking treemap.py check --cap-only
├── scripts/
│   └── treemap.py               # deterministic helper (Python 3.11+, stdlib only)
├── tests/
│   └── test_treemap.py          # unittest suite (fixtures built in temp dirs)
├── README.md
└── LICENSE                      # MIT
```

Deterministic/semantic split: **`treemap.py`** does everything mechanical (walk,
boundary detection, dependency graph, marker I/O, caps, drift). **The model** (skill)
fills only the semantic fields (purpose, abstractions, gotchas) and delegates all
writes to `treemap.py write-block`.

Scripts are always invoked as `${CLAUDE_PLUGIN_ROOT}/scripts/treemap.py`, never via
cwd-relative paths.

## Component: `treemap.py`

Python 3.11+, stdlib only, single file. Global flags: `--root <dir>` (target repo root,
default cwd), `--json` where applicable. Exit codes: `0` OK, `1` drift/cap violation
(check), `2` usage or integrity error (e.g. corrupted markers).

### Subcommands

- **`scan [--json]`** → module discovery + status. JSON shape:
  ```json
  {
    "root": "/abs/path",
    "modules": [
      {
        "path": "src/api",
        "kind": "framework-dir | package | monorepo-member",
        "source_count": 14,
        "depends_on": ["src/core"],
        "used_by": ["src/routes"],
        "has_manual_content": true,
        "block_present": true,
        "over_cap": false
      }
    ],
    "root_map": {"present": true, "entries": [{"path": "src/api", "pointer": "src/api/CLAUDE.md"}]},
    "ignored": ["node_modules", "dist"]
  }
  ```
- **`check [--json] [--cap-only] [--path <file>]`** → drift report. JSON shape:
  ```json
  {"drift": {"new": [], "renamed": [], "removed": [], "dead_pointers": [],
             "orphan_nested": [], "cap_violations": []}}
  ```
  Exit 1 if any drift bucket is non-empty. `--cap-only` restricts to cap checking
  (used by the hook); `--path` restricts cap checking to one file. Human-readable
  (non-`--json`) output is the tabular report: columns `type | path | detail | suggested-action`.
- **`write-block --path <dir> [--kind module|root-map] [--content-file <f>]`** →
  replaces **only** the text between markers (content from `--content-file` or stdin).
  Creates `CLAUDE.md` if absent. If the file exists without markers, appends the block
  at the end, preserving all existing text. Refuses (exit 2) on corrupted or duplicated
  markers. Verifies caps post-write and warns on stderr if exceeded (write still succeeds;
  the caller reports it).
- **`read-block --path <dir> [--kind module|root-map]`** → prints the current block
  body (exit 0), empty output + exit 0 if no block, exit 2 on corrupted markers.

### Marker grammar (exact strings)

Module block, in `<module>/CLAUDE.md`:
```
<!-- BEGIN treecode (auto) — do not edit inside this block -->
...generated content...
<!-- END treecode (auto) -->
```
Root map block, in `./CLAUDE.md`:
```
<!-- BEGIN treecode:map (auto) -->
...generated map...
<!-- END treecode:map (auto) -->
```
Detection matches on the `<!-- BEGIN treecode` / `<!-- END treecode` line prefixes so
the em-dash suffix is not load-bearing. Marker names are configurable
(`markers.module`, `markers.root`) but the defaults above are the contract.
Integrity rules: exactly zero or one block per file per kind; BEGIN without END,
END without BEGIN, or duplicate blocks → integrity error (exit 2, no write).

### Ignore handling

1. If the target is a git repo and `git` is available: enumerate candidate files with
   `git ls-files --cached --others --exclude-standard` (accurate `.gitignore` semantics).
2. Fallback (no git): best-effort `.gitignore` parsing with `fnmatch` (supported subset:
   name patterns, `dir/` patterns, `*` globs; documented limitation).
3. `.claudeignore` (root-level) and config `ignore_globs` are always applied on top,
   via `fnmatch` against repo-relative paths.
4. Symlinks pointing outside the repo root are never followed.

### Boundary detection (PRD §9)

In priority order, over non-ignored directories:
1. **Package markers**: any dir containing one of `boundaries.package_markers` is a module
   (`kind: "package"`). The repo root itself is never a module.
2. **Monorepo globs**: dirs matching `boundaries.monorepo_globs` → one module each
   (`kind: "monorepo-member"`).
3. **Framework dirs** (skipped when `stack_aware=false` or `--generic`): dirs matching
   `boundaries.framework_dirs`, plus dirs under `src/` with ≥ `min_sources` source files
   (`kind: "framework-dir"`).
4. **Merge**: candidate dirs with fewer than `min_sources` source files collapse into
   their parent candidate (or are dropped if no parent candidate exists).
5. **Depth cap**: no module deeper than `max_depth` path segments from the repo root.
6. Nested candidates inside an already-selected module are absorbed by it (one module,
   one CLAUDE.md) unless they are themselves package-marker modules.

"Source file" = extension in a fixed internal set (`.py .ts .tsx .js .jsx .svelte .go
.rs .java .rb .php .c .cc .cpp .h .hpp .cs .kt .swift`).

### Dependency graph (best-effort)

- Python: `ast.parse` per file; `import`/`from` targets resolved to sibling module paths.
- JS/TS/Svelte: regex over `import ... from '...'` / `require('...')`; only relative
  imports resolved to module paths.
- Any parse error → skip file silently. `depends_on`/`used_by` are the module-level
  edges derived from file-level imports.

### Drift computation (stateless)

Inputs: current boundary detection vs. what exists on disk (nested CLAUDE.md blocks +
root map entries). Buckets:
- `new`: detected module with no nested block.
- `orphan_nested`: nested CLAUDE.md with a treecode block in a dir that is no longer a
  detected module.
- `renamed`: pairing of one `new` + one `orphan_nested` whose block bodies are ≥ 60%
  similar (`difflib.SequenceMatcher.ratio()`); reported as a suggestion, the paired
  entries are removed from `new`/`orphan_nested`.
- `removed`: root-map entry whose module is gone and has no rename pairing.
- `dead_pointers`: root-map entry pointing to a nonexistent CLAUDE.md.
- `cap_violations`: root file over `caps.root` lines; nested block-bearing files over
  `caps.nested` lines counted on total file lines; anything over `caps.hard_max` is
  flagged regardless of kind.
Nothing is ever deleted automatically; every bucket maps to a `suggested-action`.

### Configuration

Read from `treemap.config.json` at the target repo root; hardcoded defaults when absent
(defaults = PRD §8 verbatim):

```json
{
  "caps": { "root": 80, "nested": 60, "hard_max": 200 },
  "boundaries": {
    "min_sources": 3,
    "max_depth": 4,
    "package_markers": ["pyproject.toml", "package.json", "go.mod", "Cargo.toml", "pom.xml"],
    "framework_dirs": ["src/routes", "src/lib", "app/api", "src/app"],
    "monorepo_globs": ["packages/*", "apps/*", "libs/*"]
  },
  "ignore_globs": ["**/node_modules/**", "**/.venv/**", "**/dist/**", "**/build/**", "**/__pycache__/**"],
  "markers": { "module": "treecode", "root": "treecode:map" },
  "generated_language": "en",
  "hooks": { "cap_guard": "warn", "instructions_loaded_log": false },
  "stack_aware": true
}
```

Unknown keys are ignored; partial configs are deep-merged over defaults.

## Component: skill `tree-mapper`

`skills/tree-mapper/SKILL.md`, registered as `treecode:tree-mapper`. Concise
`description` (name+description load at startup; body is lazy). Workflow:

1. Run `treemap.py scan --json` → modules, drift-relevant state, cap violations.
2. Present the **plan** (modules to create/update, counts). Ask for confirmation when
   more than **15** files would be touched (`--dry-run` stops here).
3. Per module: read key sources, fill the semantic template (below), write via
   `treemap.py write-block --path <dir>` (never edit CLAUDE.md files directly).
4. Update the root map block via `write-block --path . --kind root-map`.
5. Run `treemap.py check` → final report (files written, caps OK, residual drift).
6. Never commit; print the modified-file set and suggest a commit message.

Semantic fields are written in `generated_language`. Nested template (inside markers):

```markdown
# <module-name> — <one-line purpose>

Responsibility: <what this module owns; what it must NOT do>
Key abstractions: <classes/interfaces/entrypoints that matter>
Depends on: <internal modules it imports>
Used by: <internal modules that import it>
External deps of note: <libs with non-obvious usage>
Conventions: <module-local rules>
Gotchas: <non-obvious behaviors, footguns>
```

Root map template (inside `treecode:map` markers):

```markdown
## Codebase map
- `src/api/`  — HTTP handlers  → src/api/CLAUDE.md
```

## Component: commands

- **`/treecode:map-tree [path] [--dry-run] [--force] [--generic]`** — invokes the
  `tree-mapper` skill. `path` limits the sync to a subtree. `--force` regenerates
  blocks even for unchanged modules. `--generic` passes `stack_aware=false` behavior
  to the scan.
- **`/treecode:map-drift [path]`** — audit only: runs `treemap.py check`, prints the
  drift table, writes nothing. Exit code ≠ 0 on drift (CI-friendly).

## Component: hooks

`hooks/hooks.json`, canonical keyed-object schema:

- **`PostToolUse`** on `Write|Edit`, matcher on `CLAUDE.md` filenames → runs
  `cap-guard.sh`, which calls `treemap.py check --cap-only --path <file>` and prints a
  warning if over cap. **Fail-open**: never blocks, exit 0 always (`hooks.cap_guard:
  "warn"`); a future `strict` mode may block, out of scope for v0.1 behavior beyond
  reading the config value.
- **`InstructionsLoaded`** (all `load_reason`s) → appends a line to
  `.claude/treecode-instructions.log` **only if** `hooks.instructions_loaded_log` is
  `true` in the target repo config; otherwise no-op.

## Edge cases (behavioral contract)

- Pre-existing CLAUDE.md without markers → append block at the end; never claim
  ownership of the rest of the file.
- Corrupted/duplicated marker block → refuse to write, report, exit 2. Never guess.
- Renamed module → `renamed` drift row with suggested move; never auto-delete.
- Orphan nested file → `orphan` drift row with suggested removal; never delete.
- Huge repo → `--dry-run` plus the >15-files confirmation gate.
- Symlinks out of the repo root → not followed.
- `generated_language != en` → semantic fields in that language; markers unchanged.

## Acceptance criteria (v0.1)

1. On a fixture repo (Python `src/` + SvelteKit-style `frontend/`), `/map-tree` creates
   nested CLAUDE.md files in the expected modules and a map block in the root, all
   under caps.
2. Re-running `/map-tree` with no code changes → **zero diff** (idempotency).
3. Human text outside markers survives a re-run.
4. No `@path` import is ever written to wire the tree.
5. `/map-drift` on a repo with a renamed module → exit ≠ 0 and a correct `renamed` row.
6. Manually editing a CLAUDE.md over cap triggers the hook warning; the write is not
   blocked.
7. `treemap.py` runs with no `pip install` on a clean Python 3.11.
8. No file in `ignore_globs`/`.gitignore` is created or modified.
9. `claude plugin validate .` passes; `/plugin install treecode@iu3qez-tools` from the
   repo registers commands, skill, and hooks.

## Deferred to v0.2

- `module-cartographer` subagent + parallel fan-out (M5).
- `--with-rules` generation of `.claude/rules/<module>.md` (M6).
- `strict` cap-guard mode that actually blocks writes.
